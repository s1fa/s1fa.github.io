Building A Blog Websiteüåè#

Based on Hugo the hugo-PaperMod theme.

Site demo:

Installing hugo#

Follow Hugo Docs’s - Quick Start guide to install Hugo.

macOS#

brew install hugo

Windows#

scoop install hugo-extended

Git must be installed as well.

Creating a site#

Execute the following command in the terminal. It will create the directory structure for your project in the blogwebsite directory.

hugo new site blogwebsite --format yaml

Installing PaperMod Theme#

  • Themes reside in blogwebsite/themes directory.
  • PaperMod will be installed in blogwebsite/themes/PaperMod
cd blogwebsite

git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

Configuring the site:#

Edit the config file#

With the editor, open the site configuration file (hugo.yaml) in the root of the project.

# use vscode to edit the config file

vi hugo.yaml

Set a Favicon for Your Site#

  1. Download a Favicon:
    • Visit the favicon website and download a favicon.
    • Unzip the downloaded favicon file.
    • Copy the favicon.ico icon and paste it into your project’s static directory at /websiteblog/static/favicon.ico.
  2. Modify the baseof.html File:
    • Copy the /websiteblog/themes/PaperMod/layouts/_default/baseof.html file, maintaining the directory structure.
    • Paste it into your project directory at /websiteblog/layouts/_default/baseof.html.
    • Modify the baseof.html file as shown below:
<!DOCTYPE html>
<html lang="{{ site.Language }}" dir="{{ .Language.LanguageDirection | default "auto" }}">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{{ .Title }}</title>
    <!-- Favicon links -->
    <link rel="icon" type="image/x-icon" href="{{ .Site.Params.assets.favicon }}">
    {{- partial "head.html" . }}
</head>

touch /content/archives.md /content/search.md

Adding content to the BlogWebsite#

Add a new page to your site.

hugo new content posts/my-first-post.md

Hugo created the file in the content/posts directory. Open the file with your editor.

+++
title = 'My First Post'
date = 2024-01-14T07:07:07+01:00
draft = true
+++

## Introduction

This is **bold** text, and this is *emphasized* text.

Visit the [Hugo](https://gohugo.io) website!

Notice the draft value in the front matter is true. By default, Hugo does not publish draft content when you build the site. Learn more about draft, future, and expired content.

Add some Markdown to the body of the post. When satisfied with your new content, set the front matter draft parameter to false. And run the command:

hugo

Deploying the Website#

  1. Initialize a Git Repository:

    • Navigate to the project’spublic directory.

    • Initialize a git repository if you haven’t already:

      git init

  2. Commit Your Changes:

    Add all your files to the repository and Commit the changes

    git add .
git commit -m "Initial commit"

  3. Create a GitHub Repository:

    • Go to GitHub and create a new repository.
    • Do not initialize the repository with a README, .gitignore, or license file.

  4. Push Your Local Repository to GitHub:

    • Link your local repository to the GitHub repository you just created:

      git remote add origin https://github.com/yourusername/yourrepositoryname.git

    • Push your changes to the GitHub repository:

      git push -u origin main

  5. Configure GitHub Pages:

    • Go to the repository settings on GitHub.
    • Scroll down to the “GitHub Pages” section.
    • Under “Source,” select the main branch and the /root folder.
